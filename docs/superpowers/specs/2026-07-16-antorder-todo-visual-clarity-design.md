# 蚁序待办视觉清晰度重设计

## 背景

蚁序待办已经具备完整的三栏工作台、快速记录、分类筛选、日期筛选和多种列表视图，但当前浅色界面存在三个直接影响阅读的问题：

1. 全局导航、上下文侧栏和主工作区使用接近的灰色与半透明表面，区域边界主要依赖浅描边和阴影，整体显得灰、软、模糊。
2. 次级文字、日期、备注、工具栏图标和禁用控件对比度不足，长列表扫描效率偏低。
3. 选中、优先级、进度和逾期状态同时使用蓝、橙、红等多种强调方式，语义竞争导致视觉噪音。

本设计在不改变待办内容、导航顺序、数据模型和核心交互的前提下，统一视觉层级和状态语义。

## 设计方向

整体采用“Linear 主导的 macOS 原生生产力工具”：

- Linear：页面层级、克制配色、紧凑列表和交互反馈。
- Apple：系统字体、控件状态、焦点环、键盘操作和原生窗口感。
- Things 3：待办语义、日期表达、完成反馈和空状态。
- Notion：标题、备注和元数据之间的信息密度平衡。

四套参考不做外观拼接。Linear 决定整体骨架，Apple 决定控件行为，Things 3 决定待办语义，Notion 只用于校准信息密度。

## 目标

1. 不依赖阴影也能清楚识别三栏结构。
2. 主文字和正文级次级文字适合长时间阅读。
3. 一个状态只使用一套视觉信号，红色恢复为异常色。
4. 保留紧凑列表和自然语言“时间 + 标题”阅读顺序。
5. 五套皮肤共享同一布局、表面、字号、圆角和状态规则，只改变强调色。
6. 深色模式与浅色模式具有相同的信息层级。

## 非目标

- 不调整“今日推进、未完成、等待反馈、本周固定、已完成、全部待办”的顺序和业务含义。
- 不改变右键菜单承载编辑、删除和状态操作的策略。
- 不新增优先级、进度或日期字段。
- 不重构 SQLite、同步、AI 解析或任务存储逻辑。
- 不把页面改成卡片式仪表盘。
- 不在本轮重设计手记、凭证和设置页面的具体内容布局；它们只继承共享主题令牌。

## 全局视觉基线

### 表面层级

浅色模式使用稳定的中性表面：

| 令牌 | 建议值 | 用途 |
| --- | --- | --- |
| `canvas` | `#F4F5F7` | 窗口基础背景 |
| `moduleRail` | `#EEF0F3` | 全局模块导航 |
| `contextSidebar` | `#F7F8FA` | 待办分类与日期侧栏 |
| `contentSurface` | `#FFFFFF` | 主工作区 |
| `contentAltSurface` | `#F8F9FB` | 工具栏或局部辅助区域 |
| `listRowHover` | `#F4F6F8` | 行悬停 |
| `listRowSelected` | `#EAF1FF` | 行或导航选中 |
| `hairline` | `#D9DEE7` | 1px 分隔线和必要描边 |

规则：

- 三栏只使用背景差和 1px 分隔线建立层级。
- 页面区块、任务行、侧栏导航和日历不使用投影。
- 阴影仅用于菜单、浮层、模态窗口和拖拽预览。
- 禁止使用透明白色叠层模拟表面层级。
- 同一水平区域内不出现两层以上圆角容器。

深色模式不逐字复用浅色值，但保持相同的亮度关系：模块导航最暗，上下文侧栏次之，主工作区最清晰。主要相邻表面必须在不使用阴影时仍可区分。

### 文字层级

| 内容 | 字号/字重 | 浅色建议值 |
| --- | --- | --- |
| 页面标题 | 15px / 600 | `#20242A` |
| 任务标题 | 14px / 600 | `#20242A` |
| 导航标题 | 13px / 600 | `#20242A` |
| 时间信息 | 12px / 500 | `#596270` |
| 任务备注 | 12px / 400 | `#596270` |
| 导航说明 | 11px / 400 | `#6B7481` |
| 非必要元数据 | 11px / 400 | `#7A8492` |

规则：

- 使用 SF Pro 与 PingFang SC 系统字体，不引入自定义字体。
- 字间距统一为 0。
- `textPrimary` 和正文级 `textSecondary` 对背景的对比度不低于 4.5:1。
- `textMuted` 仅用于可以忽略的元数据，不用于任务备注、日期或控件标签。
- 禁用态降低强调程度，但文字仍需可辨，不能只通过整体低透明度处理。

### 色彩语义

| 语义 | 用途 |
| --- | --- |
| `accent` | 选中、键盘焦点、主要操作 |
| `danger` | 逾期、错误、阻塞 |
| `warning` | 需要关注但尚未失败 |
| `success` | 完成、成功 |
| 中性灰 | 普通状态、普通优先级、辅助图标 |

规则：

- 每个组件最多使用一个强调色体系，语义异常色除外。
- 红色不用于普通未完成数量、普通优先级和装饰性竖线。
- 不通过增加更多颜色区分中低优先级；普通任务保持中性。
- 五套皮肤只替换 `accent` 与 `accentSoft`，不改变危险、警告和成功语义。
- `accentWarm` 不再作为第二主要操作色；主要操作统一使用当前皮肤的 `accent`。

## 页面结构

现有三栏和宽度保持不变：

```text
WorkspaceShell
├── ModuleRail             184px
├── ContextSidebar         264px
└── WorkspaceContent       flexible
```

- `ModuleRail` 使用最明确的导航背景。
- `ContextSidebar` 使用较轻背景，负责当前模块筛选和日期。
- `WorkspaceContent` 使用白色或深色模式下的主内容表面。
- 顶栏和局部工具栏以分隔线归属到对应区域，不做悬浮卡片。

## 模块导航与待办侧栏

### 模块导航

- 保留品牌、空间名称以及待办、手记、凭证、设置入口。
- 选中项使用 `accentSoft` 背景、`accent` 图标和文字。
- 未选中项使用主文字与中性图标，不使用低对比度整体透明。
- 悬停只增加中性背景，不增加描边或阴影。
- 圆角统一为 6px。

### 待办分类

保留已经确认的顺序和图标：

1. 今日推进：`scope`
2. 未完成：`circle.dashed`
3. 等待反馈：`hourglass`
4. 本周固定：`repeat.circle`
5. 已完成：`checkmark.circle.fill`
6. 全部待办：`tray.full.fill`

规则：

- 选中项只使用当前皮肤的强调色，不再同时出现橙色竖线、蓝色描边和红色数量。
- 数量徽标默认使用中性色；只有数量本身表示逾期时才使用 `danger`。
- 导航标题与说明保持现有内容，但提高说明文字对比度。
- 行高和点击区域保持稳定，hover、选中和数量变化不得改变布局尺寸。

## 任务列表

### 阅读顺序

任务主行保持自然语言顺序：

```text
[完成框] [单一状态图标] [时间] [标题]
                           [备注]
```

- 时间与标题在同一阅读流中，不拆成表格列。
- 标题是最强信息；时间和备注是可持续阅读的次级信息。
- 备注与标题起始位置对齐，最多显示两行；完整内容通过详情或系统提示查看。

### 密度

- 无备注任务行最小视觉高度约 40px。
- 带备注任务行约 54px，根据两行备注自然增高。
- 完成框视觉尺寸约 18px，实际交互区域保持至少 38px。
- 行内水平间距使用 6、8、12 的节奏，不使用 16px 以上的大面积卡片留白。
- 行间使用 1px 低对比分隔线或等效的稳定留白，不使用独立卡片。

### 状态图标

- 普通任务最多展示一个中性状态图标。
- 进度与优先级同时存在时，按“异常优先、进度其次”的规则合并为一个主信号。
- 高优先级且逾期时只显示一个 `exclamationmark.circle.fill`，不再叠加闪电、旗帜和红色竖线。
- 协作、等待等业务状态使用线型 SF Symbol，并保持统一视觉尺寸。

### 逾期

- 使用红色日期作为主要信号。
- 必要时增加一个红色警告图标。
- 取消红色背景、红色行边框和左侧连续红色竖线。
- 普通标题保持主文字颜色，避免整行变红。

### 行交互

- hover 使用 `listRowHover`，持续时间约 120ms。
- 选中使用 `listRowSelected`，不使用阴影。
- 点击完成框只改变完成状态，不触发整行详情。
- 编辑、删除、调整状态和优先级继续通过右键菜单处理。
- 不在 hover 时显示一排操作按钮。
- 完成反馈使用轻量淡出或文字减淡，并遵循“减少动态效果”设置。

## 局部工具栏

- 工具栏高度维持紧凑，不新增第二层标题。
- 搜索框高度 32px、圆角 8px，默认使用明确描边。
- 聚焦时使用 1.5px `focusRing`，不得通过投影表达焦点。
- 紧凑、分组、看板、四象限使用同一分段控件家族。
- 当前模式使用 `accentSoft` 背景和 `accent` 内容，不使用突兀的整块蓝色按钮。
- 图标按钮使用 SF Symbols，视觉尺寸和点击区域固定，避免 hover 引发布局抖动。

## 快速记录

- 折叠态是主内容区中的平面输入条，不是悬浮卡片。
- 默认圆角 8px、1px 边框、无阴影。
- 聚焦态使用 1.5px `focusRing`，左侧不再显示第二种强调色竖线。
- “记录”按钮仅在内容有效时使用 `accent`；禁用态使用中性背景和可辨文字。
- 展开后的优先级、进度、日期和固定任务控件保持原有功能，但减少胶囊标签数量。
- AI 解析结果作为次级反馈，不抢占任务标题输入的视觉权重。

## 快速日期与小日历

- 快速日期按钮使用固定尺寸；普通日期不使用独立白色卡片背景。
- 只有选中日期使用 `accentSoft`，今天使用强调色文字或细描边。
- 年份和月份合并到一条日历导航，避免两层胶囊控件。
- 月历主体不使用外层阴影，必要时只保留 1px 分隔线。
- 普通日期无背景，hover、今天和选中日期才显示背景。
- 事件标记最多两种：未完成使用 `accent`，全部完成使用中性或 `success`。

## 组件尺寸与状态

| 组件 | 圆角 | 最小交互尺寸 |
| --- | --- | --- |
| 图标按钮 | 6px | 32x32px |
| 导航项 | 6px | 36px 高 |
| 搜索与输入框 | 8px | 32px 高 |
| 快速记录 | 8px | 40px 高 |
| 菜单与弹窗 | 系统样式 | 系统规则 |

所有可交互组件必须定义：

- Default：稳定边界与可读标签。
- Hover：中性背景变化。
- Active：轻微压暗或缩放，不移动布局。
- Focus：明确焦点环。
- Disabled：降低强调程度但保持文字可辨。
- Selected：只使用一个强调色体系。

## 皮肤策略

`ocean`、`aurora`、`board`、`leafcutter`、`workspace` 共享全部结构令牌。各皮肤只提供可访问的 `accent` 和 `accentSoft`：

- `ocean`：清晰蓝色。
- `aurora`：克制紫色，仅作为强调色。
- `board`：低饱和洋红色，仅作为强调色。
- `leafcutter`：稳重绿色，不使用大面积黄绿或橙色表面。
- `workspace`：中性专业蓝，作为默认主题。

皮肤切换不得改变页面背景层级、控件尺寸、圆角、阴影、文字颜色等级和语义颜色。

## 与既有设计文档的关系

本设计继承 `2026-07-01-antorder-command-workspace-redesign.md` 的工作台信息架构和“皮肤只改变配色”原则，并覆盖其中以下视觉规则：

- 逾期不再使用红色背景、边框和文字共同表达，改为红色日期与必要的单一警告图标。
- 年份和月份由分层控件收敛为同一条导航。
- 页面表面不再依赖半透明白色和行阴影建立层级。
- 待办行从多图标状态表达收敛为单一主状态图标。

业务行为、全局搜索和工作台结构不受影响。

## 验收标准

### 视觉

- 关闭任务行和页面区块阴影后，三栏区域仍清楚可辨。
- 主文字和正文级次级文字对比度不低于 4.5:1。
- 一个选中组件只出现一种强调色。
- 普通任务行最多出现一个彩色语义图标。
- 逾期任务没有红色背景、红色边框或连续红色竖线。
- 快速记录、搜索框和日历不再形成卡片套卡片。
- 页面在 1280、1440 和 1920 宽度下无文字重叠或控件抖动。

### 交互

- 任务右键菜单功能保持完整。
- 完成框、导航项、日期和工具栏控件保留现有点击行为。
- hover、focus、active、disabled 和 selected 五种状态可明确区分。
- 深浅色模式和五套皮肤切换不改变布局尺寸。
- 减少动态效果启用时，不出现依赖动画才能理解的反馈。

### 回归验证

实施阶段必须运行：

```bash
scripts/run_quality_checks.sh
swift build
swift run loop-selftest
swift run loopctl doctor
git diff --check
```

其中 `loop-selftest` 和 `loopctl doctor` 在 Loop Engineering 控制平面根目录执行；DailyTodos 自身检查在隔离工作区执行。

视觉验证至少覆盖：

1. 浅色与深色模式。
2. 五套皮肤。
3. 1280x800、1440x900、1920x1080 三种窗口尺寸。
4. 普通、逾期、高优先级、等待、完成和带备注任务。
5. 导航、任务行、搜索、快速记录、日历的 hover、focus、disabled 和 selected 状态。

## 设计自检

- 没有改变业务数据或核心交互。
- 没有新增卡片式页面结构。
- 没有用更多颜色解决状态识别问题。
- 没有把必要正文降级为低对比度文字。
- 没有让皮肤继续改变布局语言。
- 没有使用 TBD、TODO 或未决占位项。
